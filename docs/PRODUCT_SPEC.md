# Product Spec — AetherTwin Fairy OS

## Vision

AetherTwin Fairy OS is a magical desktop companion layer where AI agents are embodied as fairies. Each fairy has a vessel, personality, memory, skill, strengths, weaknesses, permissions, and growth path.

## Core user promise

> Your computer becomes a fairy sanctuary where specialized companions help you build, apply, remember, protect, ship, and express your work.

## Primary modules

### 1. Sanctuary

Shows the active fairy vessel, Genesis profile, strengths, weaknesses, shadow risk, XP, and skill.

### 2. Genesis Hall

Creates new fairies with:

- name
- avatar glyph/body reference
- archetype
- origin
- sacred purpose
- strengths
- weaknesses
- shadow risk
- skills
- avatar vision sources
- allowed controls
- forbidden actions
- memory scope
- growth path

### 3. Skill Registry

Each fairy owns one or more skills. Skills define:

- inputs
- outputs
- commands
- permission tier
- future tool adapters

### 4. Fairy Council

Multiple fairies give short perspectives. The system produces a synthesis. This prevents one fairy’s weakness from dominating.

### 5. AetherSense

Simulated telepathy layer. It routes pasted signals to a fairy:

- job post -> Fae
- code/error -> Glint
- visual/aesthetic -> Stella
- planning overwhelm -> Hope
- unknown fragment -> Lyra

Future desktop version should use consent-based:

- selected text
- clipboard
- screenshot region
- active window title
- current folder
- browser extension context

### 6. Memory Tree

User-editable memory branches:

- career
- projects
- art style
- games
- memory
- permissions
- social
- shipping
- dreams

### 7. Permission Grimoire

Visible permissions per fairy. No high-risk action happens silently.

### 8. Agent Handoff

Copyable brief and docs so any agent can continue the project.
